38  *
39  * Function Name : LIN_DRV_Init
40  * Description : This function initializes a LIN Hardware Interface for operation.
41  * This function will initialize the run-time state structure to keep track of
42  * the on-going transfers, ungate the clock to LIN Hardware Interface, initialize the
43  * module to user defined settings and default settings, configure the IRQ state
44  * structure and enable the module-level interrupt to the core, and enable the
45  * LIN Hardware Interface transmitter and receiver.
46  * The following is an example of how to set up the lin_state_t and the
47  * lin_user_config_t parameters and how to call the LIN_DRV_Init function
48  * by passing in these parameters:
49  * lin_user_config_t linUserConfig
50  * linUserConfig.baudRate = 9600
51  * linUserConfig.nodeFunction = SLAVE
52  * linUserConfig.autobaudEnable = true
53  * linUserConfig.timerGetTimeIntervalCallback = (lin_timer_get_time_interval_t) l_ifc_timerGetTimeIntervalCallbackHandler
54  * lin_state_t linState
55  * LIN_DRV_Init(instance, (lin_user_config_t *) &linUserConfig, (lin_state_t *) &linState)
56  *
57  * Implements : LIN_DRV_Init_Activity
58  *END**************************************************************************/

125  * Function Name : LIN_DRV_SendFrameDataBlocking
126  * Description : This function sends data out through the LIN Hardware Interface using
127  * blocking method. This function will calculate the checksum byte and send it with the
128  * frame data. Blocking means that the function does not return until the transmission is
129  * complete. This function checks if txSize is in range from 1 to 8.

133  * will return STATUS_SUCCESS. If not, it will return STATUS_TIMEOUT.
134  *

239  * check if rxSize is in range from 1 to 8. If not, it will return STATUS_ERROR. This function
240  * also checks if the isBusBusy is false, if not it will return STATUS_BUSY.

LIN_DRV_SendFrameData
status_t LIN_DRV_SendFrameData(uint32_t instance, const uint8_t *txBuff, uint8_t txSize)
Sends frame data out through the LIN Hardware Interface using non-blocking method. This enables an a-sync method for transmitting data. Non-blocking means that the function returns immediately. The application has to get the transmit status to know when the transmit is complete. This function will calculate the checksum byte and send it with the frame data. The function will return immediately after calling this function. If txSize is equal to 0 or greater than 8 or node's current state is in SLEEP mode then the function will return STATUS_ERROR. If isBusBusy is currently true then the function will return LIN_BUS_BUSY.
